# EXEC_PLAN.md

## Current Slice

ANSI-native Details diff rendering.

## Goal

- show Details diff/log payloads as bounded native Git command output
- preserve Git ANSI colors in the TUI Details panel without semantic diff row
  parsing on every render
- keep large commit and folder diff previews capped so a huge output does not
  monopolize UI refresh
- continue using the existing synthetic large repository for empirical perf
  checks; do not regenerate it

## Vertical Slice

1. Backend output
- use `git diff --color=always` for tracked Files Details unstaged/staged
  patches
- use `git show --color=always` for commit details and Commit Files patch
  previews
- keep untracked Files Details patches generated by the existing safe formatter,
  because plain `git diff` does not emit untracked file contents
- keep Details diff output byte-limited and append deterministic truncation
  notices
- pass pathspecs with Git literal pathspec magic so special characters remain
  literal

2. UI rendering
- render Files, Branches, Commits, and Commit Files Details text through the
  shared ANSI span projection
- remove per-render semantic diff row classification from Details diff output
- keep rendering pure: UI reads only `AppState` text and projects ANSI spans

3. Tests
- replace diff-role Details tests with native-output and ANSI-span assertions
- keep literal pathspec and bounded-output backend coverage
- update performance-suite baselines to use the same colorized Git diff/show
  commands

4. Documentation
- document that Details uses ANSI-aware native Git output for diff/log payloads

5. Validation
- run `cargo fmt`
- run `cargo clippy --workspace --lib --bins -- -D warnings`
- run `cargo test`
- run the relevant large-repo perf case against the existing repo without
  `--regenerate`
